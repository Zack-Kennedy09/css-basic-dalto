Outline

Es un shortened property (propiedad acortada) porque es el conjunto de otras propiedades, como por ejemplo tenemos border-color, border-widht, border-style. Y esas propiedades se abrevian colocando solo border.
Abreviando lo anterior seria: border: 1px solid red;
de esa forma se reduce codigo y es una buena practica

Outline es como un border pero sin afectar a las demas cajas, no ocupa un espacio real en el el DOM. El DOM es el Document Object Model (Modelo de Objetos del Documento), estos son todos los elementos que están dentro de nuestra página, se puede decir que si forma parte del DOM pero no afecta las posiciones, los tamaños, no cambia nada, solamente resalta; es como una especie de resaltador

Por ejemplo:

Agregamos dos divs con las clases "clase1 y clase2"
Luego le damos estilo. *Esto en CSS*:

.clase1 {
width: 100px;
height: 100px;
background: red;
}

.clase2 {
width: 100px;
height: 100px;
background: yellow;
}


Al momento de darle las mismas propiedades a diferentes clases, se puede acortar colocando las dos clases en la misma posicion del selector, pero separados por una coma ","

.caja3, caja4 {
width: 200px;
height: 200px;
background: red;
}


No se ven ambos cuadros??


Pero como yo voy a darle otras propiedades, sigo escribiendo en las dos clases por separado.
Lo que vamos a hacer es ver como una caja afecta a la otra caja...
las centramos, dandole un "margin: auto" a ambas.

El margin:auto funciona: arriba y abajo lo que se defina o lo que ya está y a los lados es 50 y 50 (el tamaño total del contenedor menos la mitad de la caja de margin left y lo mismo con margin right). Es un calculo automatico que hace el navegador cuando le decimos margin:auto. Esto funciona con los elementos que son bloques y no están posicionados (mas adelante se habla bien de eso)



Que pasa si a la caja de arriba le doy estos valores:
border: 10px solid blue



Logicamente adquiere un borde azul, ahora esto está ocupando un lugar (le esta sacando un lugar al rojo).
Recordando que la propiedad "box-sizing: border-box;" se agregó en el normalize, esto por defecto tiene la propiedad de "box-sizing: content-box;"...

box-sizing: content-box;

.


De ésta manera, si le asignamos 150px a la caja, le suma a esos 150 los 10px de cada lado que corresponde a los bordes. Asi que serian 170px por 170px el tamaño total de la caja (caja roja). A la vez que el borde ocupa el espacio, la caja amarilla la mueve hacia abajo la cantidad de pixeles que ocupa verticalmente la caja roja

Si reemplazamos la propiedad de border por la propiedad outline, va hacer el mismo efecto de borde pero no ocupará espacio ni moverá las cajas

outline: 10px solid blue;


.


No mueve la caja amarilla ni la roja. Asi que es para localizar/marcar el objeto
El outline es lo que tienen los campos de texto (los inputs)

Cuando hago click en el input se crea un borde, ese borde es un outline

Para comprobarlo, le sacamos la propiedad al input:
input type="text" style="outline: none !important"




Ahora si le sacamos la propiedad de "box-sizing:box-content;", la propiedad de outline se aplicará por fuera de la caja pero no ocupara espacio que afecte a las cajas:

Sin el box-sizing:box-content;
